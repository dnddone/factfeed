# 0006. Claude fact generation + typographic images

**Status:** Accepted
**Date:** 2026-07-03

## Context

Facts and their imagery are generated on the backend and persisted. Two choices:
how to produce fact text, and how to produce the accompanying image.

For images, the key economic insight is that **cost is per-post, not per-view**:
an image is generated once at fact-creation time and its URL is stored, so every
future swipe just reads it. Total image spend ≈ (number of posts) × (price per
image) — a one-time trickle, not a per-traffic cost. Options ranged from free
(programmatic typographic cards, stock photos) to paid AI generation
(FLUX schnell ~$0.003/image up to premium models ~$0.04–0.17).

## Decision

- **Fact text**: generated by **Claude** (Anthropic SDK), in batches, with
  content-hash dedupe before persisting.
- **Images**: **typographic cards** — the fact rendered on a gradient via
  `@vercel/og`. The `Post.imageSource` field records the method so we can mix in
  other sources later without a schema change.

## Consequences

- **Why typographic first**: free, fast, no external image API or key, and it
  ships the entire swipe loop with zero image cost. Visually it fits the
  Reels/Tinder card format well.
- The `imageSource` enum (`typographic | stock | ai`) is designed in from the
  start, so adding **stock** (Unsplash/Pexels) or **AI** (FLUX schnell) for a
  subset later is additive — no migration, just a new branch in the image step.
- Note: **Claude cannot generate images** — it is the text/fact generator only.
  Any future AI imagery uses a separate image model (e.g. FLUX via fal/Replicate).
- Generated assets (once we move beyond inline typographic rendering) are stored
  in **Supabase Storage** (ADR 0003) and served via CDN.
- Dedupe is content-hash based for now; embedding-similarity dedupe is a future
  upgrade if near-duplicate facts slip through.
