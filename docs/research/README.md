# research

Time-bound investigations: evaluations, spikes, comparisons, "can we even do
this" questions. Used when we don't yet know enough to propose a specific
change.

- **Folder per effort**: `research/<YYYY-MM>-<slug>/`.
- Include a `README.md` stating the question, the timebox, and — when done — the
  conclusion.
- **Outcome**: promote findings to an ADR in `decisions/` (a decision made)
  and/or a design doc in `design-docs/` (a change to build). The research folder
  stays as the record of how we got there.
